# v0.4.2 — Service Units 計畫（第一階段：安全版 scripts）

**版本標籤（規劃中）：** `v0.4.2-service-units`
**目標：** 讓 Adapter v0.4 與 Hermes Gateway 能用**固定指令**啟動、停止、檢查狀態，並為之後的「開機自動啟動」鋪路。
**不做：** v0.5 / Queue / DLQ；不改 Adapter 程式；不改 Hermes / OpenClaw 核心。

---

## 為什麼先做 scripts，不直接上 systemd

第一階段刻意**只做 shell scripts**，先不動 systemd，原因：

1. **風險低、可逆** —— scripts 只是把目前已驗證、手動在跑的指令包起來，不改變執行身分、不註冊系統服務，出錯也只影響當前 process。
2. **WSL 的 systemd 行為要先確認** —— WSL 需要 `/etc/wsl.conf` 開啟 `systemd=true` 才有 systemd；而且 user service、開機啟動、`lingering` 等行為要逐項驗證，貿然上 unit 容易踩坑。
3. **先把「啟動/停止/狀態」標準化** —— 有了穩定的 scripts，之後寫 systemd unit 時 `ExecStart` / `ExecStop` 直接指向這些 scripts 即可，過渡平滑。

## 第一階段：scripts（本次新增）

| 指令 | 作用 |
|---|---|
| `./scripts/start_adapter_v04.sh` | 背景啟動 Adapter v0.4（port 8000）。**若 8000 已被佔用會提示並結束，不會硬開第二個。** log → `logs/adapter-v0.4.log` |
| `./scripts/stop_adapter_v04.sh` | 找出 8000 上的 Adapter，顯示 pid，送 SIGTERM 溫和停止（最多等 10 秒） |
| `./scripts/status_adapter_v04.sh` | `curl /health`，確認 `version` 是否為 `0.4.0` |
| `./scripts/start_gateway.sh` | 用已驗證的 tmux 方式啟動 Hermes Gateway；**若已在跑則不重複啟動、不破壞現有 gateway**。log 鏡像 → `logs/hermes-gateway.log`（官方詳細 log 仍在 `~/.hermes/logs/gateway.log`） |
| `./scripts/status_gateway.sh` | 檢查 gateway process + tmux session + 最近的 Discord 連線 log |

> 注意：`logs/` 內的 `*.log` 已被 `.gitignore` 排除，不會進 git。

### 典型用法

```bash
cd ~/projects/hermes-openclaw-adapter

# 啟動
./scripts/start_adapter_v04.sh
./scripts/start_gateway.sh

# 檢查
./scripts/status_adapter_v04.sh
./scripts/status_gateway.sh

# 停止 Adapter
./scripts/stop_adapter_v04.sh
```

## 開機後手動恢復流程（目前限制）

> 目前 Adapter 與 Hermes Gateway **不是開機自動啟動**。WSL/機器重啟後，請依序手動恢復：

1. 開啟 WSL Ubuntu 終端機（用 `lnovo` 使用者）。
2. 確認 OpenClaw 可用：
   ```bash
   openclaw status
   ```
3. 啟動 Adapter v0.4 並確認：
   ```bash
   cd ~/projects/hermes-openclaw-adapter
   ./scripts/start_adapter_v04.sh
   ./scripts/status_adapter_v04.sh     # 應看到 version 0.4.0
   ```
4. 啟動 Hermes Gateway 並確認：
   ```bash
   ./scripts/start_gateway.sh
   ./scripts/status_gateway.sh         # 應看到 Gateway 執行中 + Discord connected
   ```
5. （可選）跑一次 PONG 端到端驗證：
   ```bash
   ./scripts/test_callback_pong.sh
   ./scripts/check_task_result.sh task-xxxx   # 應為 completed / PONG
   ```

> 提醒：若期間有**新增或變更 MCP tool**，務必重啟 Hermes Gateway 才會載入（見
> [`docs/V0.4_DISCORD_E2E_VERIFICATION.md`](V0.4_DISCORD_E2E_VERIFICATION.md) 第 8、9 節）。

## 第二階段（之後才做）：systemd unit

待 scripts 穩定後，可再進一步：

1. **確認 WSL systemd 已啟用** —— `/etc/wsl.conf`：
   ```ini
   [boot]
   systemd=true
   ```
   （改完需 `wsl --shutdown` 後重開。）
2. **建立 user-level service**（建議用 `systemd --user` + `loginctl enable-linger lnovo`，讓使用者服務在開機/登出後仍續跑），例如：
   - `hermes-adapter.service` → `ExecStart` 指向 `scripts/start_adapter_v04.sh`（或直接呼叫 uvicorn，`Restart=on-failure`）。
   - `hermes-gateway.service` → 啟動 gateway（注意 tmux 與 systemd 的取捨：systemd 管理下通常直接跑 `hermes gateway run`，不需 tmux）。
3. **開機自動啟動** —— `systemctl --user enable --now hermes-adapter hermes-gateway`。
4. **驗證**：`wsl --shutdown` 重開後，不手動操作也能 `status` 看到兩者自動恢復。

> 這一階段會獨立成 `v0.4.2-service-units` 的後續，或視情況開 `v0.4.3`。本文件先記錄方向，**本次不實作 systemd**。
